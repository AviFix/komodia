Requirements

If you install the trial on Windows XP make sure that you have either .net 2 or 3.5 installed since the demo app requires it (the SDK itself does not), on Vista/7 it's already installed.

The trial will expire on a specific date rather then on X days since install or download, if it had expired the install will fail, and if it's already installed, it will stop functioning

The WFP and LSP trial are controlled using the same instructions:

  • WFP trial can only work on Windows 8 (Metro and Desktop).
  • LSP trial can work on Windows XP and above and Windows 8 Desktop.

Evaluation guide

There are different ways to evaluate the SDK, and it depends on what you plan to do.

Proxy routing/Game acceleration

  1. You should watch this PowerPoint that shows how to redirect traffic into a proxy: Redirection into a proxy
  2. Further information about how to control the SDK from the PCController app: Komodia's Redirector installation guide#Usage.
  3. When working with the PCController app you might get an error if you are missing some OCXs, in case you do, you can read about it here: Komodia's Redirector installation guide#Required external files.
  4. API documentation if you plan to control the SDK from your application: Komodia's Redirector API Guide.
  5. Under the installation directory you have a "samples" directory with all the samples.

SSH

If the trial includes the SSH module, you can see how to configure it here: Komodia's Redirector installation guide#SSH_Module

HTTP Processing

  1. In the demo you can test the second and third option that shows how to read the HTTP data and redirect it.
  2. The source code of the demo is under the "samples" directory: VB.Net\Traffic handling samples\Show case, you also have a C++ and C# sample.
  3. API documentation if you plan to control the SDK from your application: Komodia's Redirector API Guide.
  4. To get data from the SDK you can either use the COM interface (just like the demo does): Komodia's Redirector COM framework guide which will send all data to your application or use the DLL framework (which has faster performance, but more complex to write): Komodia's Redirector DLL framework guide and will work in the context of the SDK main process.

Parental control

There are number of ways to do parental control.

Developing your own logic

  • In the demo you can test the first, second and third option that shows how to read the HTTP data and redirect it.

Using white/black lists

  • You should watch this PowerPoint that shows how to redirect traffic into a proxy: Built in parental control lists

Using Komodia's classification server

  • In the demo use the last option that shows how the SDK works with Komodia's classification server.
