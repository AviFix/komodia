2014

July

Version 2.2.9.6

  • DLL, added an ability to set/get flags and save data directly from the DLL - without using COM: Komodia's Redirector DLL framework guide#DLLSetFunctions

2013

December

Version 2.2.8.0

  • New ability to inject into HTML elements on the fly: Komodia's Redirector DLL framework guide#HTML Element injection.

Version 2.2.8.2

  • SSL Digestor, added the ability to set a custom root CA by settings the flags: (settings will be active after the SDK is restarted)
    • sslpub - To the path of the public key .pem file.
    • sslprv - To the path of the private key .pem file.
    • sslpsd - To the password of the root CA.

March

Version 2.2.4.0

  • Added ability to disable LSP usage of COM interface by setting the custom flag "nocom" to 1.
  • Added command line flag to detect other Komodia's instances (PCProxy /IsOtherInstalled)

Version 2.2.4.1

Features geared at allowing a web server to receive a configuration file, modify it and send it back:

  • Added ability to save data into a unencrypted editable text file or into a string.
  • Added ability to load the textual data back from a file or a string.

  • What is hard excluded app

  2. Central tracking server

January

Samples

New samples, and guide uploaded to: Komodia's Redirector DLL framework guide.

Service install

Service installation now writes to a log.

Interception rules

Added the ability to broadcast an update on the rules.

Switch between intercepted and non intercepted

LSP and Redirector can now change an existing connection between intercepted to non intercepted and vice versa when the rules change.

New user experience guide

Komodia's Redirector user experience guide goes over the behaviour of the Redirector and how it affects the end user experience.

2009

December

Toggling working mode

When toggling between disable and enable mode the Redirector will force every HTTP connection to go via the new mode which means that all connections that used a proxy will be redirected so they will not use proxy, and all connections that didn't use proxy will be redirected so proxy will apply to them.

LSP not breaking existing applications

When installing any kind of LSP, it can break existing network applications, this is normal behaviour for LSPs, now every Komodia's LSP have a built in feature to allow the running applications to continue working without problems (they will not be intercepted until they are closed and opened again, which is same as before).

Proxy redirecting

Added three new redirecting abilities which are controlable from the GUI and COM interface:

  1. Redirect directly to a IP/Port without doing any proxy manipulations.
  2. Redirect directly to a IP/Port from the LSP without going through the Redirector process.
  3. Redirect SSL sessions via a Socks5, the proxy will receive decrypted data and the Redirector will acquire the certificate once from the website via a direct connection.

HTTP Decoder

Added support for SDCH encoding.

Ports

All port lists now support port range, up until now it was possible to add a list of ports, so for example if you would like to enter ten ports you would have to enter them one by one, now you can enter range, e.g.: 20-30
